# Phase 1：数据采集

> **角色**：数据采集专员。按下方清单，结合执行本地 Python 脚本和 WebSearch 获取目标股票数据，整理并完善为 `data_pack_market.md`。
>
> **禁止**：不做分析判断、不做估值计算、不输出投资结论。只采集整理原始数据。

---

## 输入参数

| 参数 | 来源 |
|:-----|:-----|
| `stock_code` | 协调器提供 |
| `channel` | 协调器提供（默认：港股→港股通，A股→长期持有，美股→W-8BEN） |
| `output_dir` | 协调器提供 |

---

## 采集流程

### 第一步：运行自动化数据获取脚本
> 📍 工具：执行本地脚本 `scripts/fetch_market_data.py`
> If `yfinance` is missing, run `python -m pip install -r {strategy_dir}/scripts/requirements.txt` and retry.

请在终端中执行以下命令生成初步数据包：
```bash
python {strategy_dir}/scripts/fetch_market_data.py {stock_code} --output-dir {output_dir} --channel {channel}
```
脚本执行完毕后，读取 `{output_dir}/data_pack_market.md` 文件。该文件已包含公司的**基础信息、市场数据、五年财报、股息历史和历史价格**（即§1-§6，§11）。

### 第二步：检查、核对并补充具体缺失数据
> 📍 工具：WebSearch 或其他搜索引擎工具

请认真查阅刚生成的 `data_pack_market.md` 文件。下方罗列了完整的**"要采集的数据清单"**，请核对脚本生成的数据，并使用 WebSearch 补充所有标记为 `{待WebSearch确认}`、`{待AI补充}` 或 `⚠️缺失` 的空缺项。直接在 Markdown 文件中更新并保存。

## 具体要采集的数据内容（核对与补充清单）

### §1. 基础信息

| 项目 | 采集方式 | 取数说明/搜索关键词 |
|:-----|:---------|:-------------------|
| 股票代码 | 脚本自动填写 | 用户输入 |
| 公司名称 | 脚本自动填写 | yfinance 获取 |

### §3 ~ §5. 五年财务报表（损益表、资产负债表、现金流量表）

> ⚠️ 这部分绝大多数已由脚本通过 `yfinance` 自动排列完成。你需要核对以下核心科目是否存在。若脚本标记 `⚠️缺失`，请尝试通过 WebSearch 补充。金额单位必须是**{报表币种}百万**。

**核心监控科目（用于后续分析）：**
- 归母净利润（Net Income Applicable To Common Shares）
- 折旧摊销（Depreciation & Amortization）
- 资本支出（Capital Expenditures，取绝对值）
- 经营活动现金流（Total Cash From Operating Activities）
- 递延收入/合同负债（Deferred Revenue）（Phase 3 EV 口径需要此项）
- 存货、无形资产（含商誉）、期末现金及等价物、总有息负债、总资产、极少数股东权益

### §6. 股息与回购历史

- **股息记录**：脚本已获取列表。请通过 WebSearch 检查是否遗漏，并重点**补充**每笔股息的**类型**（中期/末期/特别）。
- **回购记录**：脚本已从现金流量表提取。如果某年份缺失，务必通过 WebSearch 确认其实际回购金额并补充。

### §10. MD&A 摘要
> 📍 WebSearch "{公司名} 年报 管理层讨论 MD&A {最新年份}" 或 "{company} annual report management discussion analysis {year}"

提取要点（每项需提供**原文片段**引用，字数≤200字）：
1. **经营回顾**与业绩归因
2. **前瞻性指引**（量化目标或明确的方向性战略表述）
3. **资本配置意图**（明确的分红/回购/资本开支投资计划）
4. **主要风险因素**

### §11. 10年历史价格摘要
> 📍 脚本已自动完成（使用 `1wk` 周期）

检查和核对包含：数据完整覆盖区间段（需覆盖10年）、10年历史最低价与最高价、年度收益率与极值等。
