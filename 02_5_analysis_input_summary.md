# Phase 2.5：分析输入摘要（Analysis Input Summary）

> **角色**：数据整合专员。将 `data_pack_market.md` 与 `data_pack_report.md` 的关键信息统一整理为 `analysis_input_summary.md`，供 Phase 3 各因子优先读取。
>
> **禁止**：不做投资结论、不做仓位判断、不输出报告正文。只整合字段、裁决口径、标注缺失与降级状态。

---

## 输入参数

| 参数 | 来源 |
|:-----|:-----|
| `output_dir` | 协调器提供 |
| `company_name` | 协调器提供 |
| `stock_code` | 协调器提供 |
| `data_pack_market.md` | Phase 1 输出（必有） |
| `data_pack_report.md` | Phase 2 输出（可选） |

---

## 目标

生成 `{output_dir}/analysis_input_summary.md`，作为 Phase 3 默认入口文件。

本摘要文件用于：
- 统一 `market/report` 两份数据包的字段口径
- 记录 A/B/C 类字段完整性
- 为后续因子输出最终应使用的前置锚定值与基础现金口径
- 显式声明本次分析采用的降级项

不在本阶段生成的内容：
- 因子3/4 才能计算出的结果性字段，例如 `FF（可自由支配现金）`、`现金保护等级`、`EV口径是否触发`
- 任何估值结论、仓位建议、价值判断

---

## 字段优先级规则

1. 当前时点市场字段：
- 当前股价
- 总市值
- 总股本
- 52周高低
- 10年历史价格

以上字段优先使用 `data_pack_market.md`。

2. 年报财务/附注字段：
- 母公司净现金
- 受限现金
- 合同负债/预收款
- 股息总额
- 支付率
- 资本化利息
- 或有负债
- 重大关联交易

以上字段优先使用 `data_pack_report.md`。

3. 发生冲突时：
- 市场字段：以 `data_pack_market.md` 为准
- 年报/附注字段：以 `data_pack_report.md` 为准
- 若仅有 `data_pack_market.md` 可用，则允许回退使用，并将状态标为 `降级`

---

## 字段状态定义

每个字段都必须标注以下 4 项：
- 字段名
- 值
- 来源
- 状态

状态仅允许使用：
- `完整`
- `降级`
- `缺失`
- `口径冲突已裁决`

## 执行要求

- 本文件生成后，Phase 3 应优先读取本摘要。
- 若本摘要不存在，Phase 3 才回退到 `data_pack_market.md` / `data_pack_report.md` 直接取数。
- 本文件只做“字段整合和口径裁决”，不进行估值、不生成投资结论。
- 若字段需要因子3/4进一步计算，需在摘要中保留其前置输入，而不是提前写入结果值。
- 若某字段仍需因子1模块〇补充，必须在本摘要中预先标注为“建议补充项”，避免后续因子自行扩散取数。
